将 Snap! 用作个人计算环境

看待编程的视角

个人计算社区和主流软件领域的分歧是观念层面的, 技术差异只是观念差异的结果。主流领域认为编程是给计算机一组指令解决某些问题, 从这个角度看, 唯一有价值的东西是结果, 一个软件只有解决某个问题才有趣, 而编程本身不有趣, 因此它应该交给专业人士来做, 其他人最好可以忽视它。个人社区认为编程是一种表达思想的方式, 软件开发的过程可以被理解为学习和记录知识的过程, 其结果顺便能够运行在计算机上, 计算机可以运行软件并且解决问题只是知识健全的结果, 但不是最重要的结果, 个人借助计算机更好的思考才是, 人是目的, 软件是强大思想的副产品。

在深入个人计算视角之后, 我逐渐远离了主流计算机生态, 开始选择来自个人计算社区的编程环境:

  • Scratch
  • Pharo
  • Squeak
  • Etoys
  • Lively

最近，我开始将 Snap! 用作个人主要的编程环境。

浏览器是新的操作系统 项目管理 - image 能力扩展 - 基于 mqtt - 消息 call, primitive, call MicroBlocks - 使用 Python 扩展能力, 获得操作系统的能力
